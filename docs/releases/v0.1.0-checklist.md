# v0.1.0 release checklist

## Scope
First public release focused on local-first RAG pilots.

## Pre-release checks
- `make f4kvs tantivy agent`
- `make test`
- `./scripts/eval_setup_public.sh http://127.0.0.1:8080`
- `./scripts/eval.sh http://127.0.0.1:8080 eval/gold/public.jsonl`

## Build artifacts
```bash
chmod +x ./scripts/release/build_binaries.sh
./scripts/release/build_binaries.sh v0.1.0
```

Expected output directory:
- `dist/v0.1.0/rag-agent_darwin_arm64`
- `dist/v0.1.0/rag-agent_darwin_amd64`
- `dist/v0.1.0/rag-agent_linux_amd64`
- `dist/v0.1.0/rag-agent_linux_arm64`
- `dist/v0.1.0/SHA256SUMS.txt`

## Manual smoke test
```bash
./dist/v0.1.0/rag-agent_darwin_arm64 -addr 127.0.0.1:8080 -data-dir /tmp/rag-release-smoke
curl http://127.0.0.1:8080/stats
```

## Release notes outline
1. Why this release exists (local, cited, evaluated RAG).
2. Technical docs (`docs/P9.md`, `docs/benchmark-public-eval.md`, `eval/README.md`).
3. Docker packaging and quick eval instructions.
4. Known limitations and best-fit deployment model.

## Publish flow
1. Create git tag `v0.1.0`.
2. Create GitHub release.
3. Upload artifacts from `dist/v0.1.0/`.
4. Paste checksums and release notes.
